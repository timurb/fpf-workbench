# Style guide

## Общие конвенции

- Язык: допускается русский, но термины/идентификаторы и пути файлов — `lowercase-kebab-case`.
- Заголовки: H1 один на страницу; дальше — H2/H3.
- Ссылки: только `[Title](relative/path.md)`, без wiki-links и без code blocks для ссылок.
- Структура: каждая рабочая страница должна иметь явные **границы контекста** (что входит/не входит) и **ссылки на источники**.

## Шаблон рабочей страницы (рекомендуется)

Используй этот каркас для страниц в `docs/work/**`:

- **Цель**
- **Контекст и границы (scope)**
- **Результат (какой артефакт хотим получить)**
- **Неизвестное / вопросы**
- **Следующие шаги (plan)**
- **Источники / evidence**

## FPF-совместимость (прагматично)

- Различай *carrier* (исходный текст/сообщение) и *description* (наша переработанная формулировка).
- Отдельно фиксируй планы (`WorkPlan`) и фактическое выполнение (`Work`), не используя планы как доказательство выполнения.

См. также: [refs/fpf/FPF-Spec.md](../refs/fpf/FPF-Spec.md).
