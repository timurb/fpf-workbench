# Contributing (docs)

## Куда добавлять новые материалы

- `docs/work/` — рабочие артефакты и страницы по отдельным направлениям (workstreams).
- `docs/_sources/` — исходные вводные (carriers): тексты промптов/брифов и другие первичные источники, на которые ссылаются страницы в `docs/work/`.

## Минимальные правила

- Markdown должен оставаться GitLab-readable.
- Ссылки — только стандартные: `[Title](relative/path.md)` (без `[[wiki-links]]`).
- Имена файлов — `lowercase-kebab-case.md`, пути — стабильные.
- Любая новая страница должна быть достижима из:
  - `docs/index.md`, и/или
  - `docs/_generated/toc.md`, и/или
  - соответствующего `docs/work/**/index.md`.

## FPF-дисциплина (лёгкая)

- Разделяй план и факт выполнения: планы/бэклог — в `docs/work/**`, а фактические логи (если появятся) — отдельными артефактами, не встраивая их в план.
- Не смешивай “объект обсуждения” и “описание/текст”: первичный ввод держи в `docs/_sources/` и ссылайся на него из рабочих страниц.

## Запреты и ограничения

- Не редактировать `refs/fpf/FPF-Spec.md` (он синхронизируется из upstream).
- Не трогать `tools/` без явного запроса.
